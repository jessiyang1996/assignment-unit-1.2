# 1 - What is the command for listing the files in the current folder (directory)?
    ls

# 2 - What is the command for changing directories?
    cd

# 3 - What is the command for going "up" a folder (to a parent directory)?
    'cd ..' or 'cd ../'

# 4 - What is a quick way to create a new folder?
    touch [fileName.fileType]
    Example: touch recess.js

    The code in the example will create a JavaScript file called "recess".

# 5 - What output would you expect when you run the command 'ls' in an empty folder? What about a folder containing files?
    If you run 'ls' in an empty folder, you will get no response back.
    If you run 'ls' in a folder containing files, you will get a list of the files in the folder.

# 6 - What is a quick way to create a new file?
    You can create a new file by using the code "mkdir [fileName]"

    example: mkdir blossoms
    In the example above, it will create a folder named "blossoms"

# 8 - Why would you run the command 'git status'?
    'git status' should be used to see any changes made locally to the files on the computer. You should run the command after making changes,
    after running commands 'git add .', 'git commit -m "[description]', 'git push' to make sure that your changes went through.

# 9 - What does the command 'git add .' do?
    The command 'git add .' will select all of your updates and bring them to the stage.

# 10 - What is the command to push your committed changes to your repo on GitHub?
    git push
